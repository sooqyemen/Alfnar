# تحديث الوارد الذكي - حفظ تلقائي للواضح + مراجعة للغامض

## ما الجديد في هذه النسخة
1. تصنيف أقوى لمحادثات واتساب الفردية والجماعية.
2. تجاهل الرسائل القصيرة والمتابعات العامة مثل: مباشر / أكد / موجود / تم.
3. عدم حفظ الرسائل الصادرة منك كعروض واردة جديدة.
4. حفظ تلقائي للعروض والطلبات الواضحة.
5. تحويل العناصر الغامضة أو الناقصة إلى قائمة مراجعة.
6. واجهة مراجعة محدثة فيها فلاتر حسب الحالة.
7. دعم أفضل لمحادثات المجموعات وكثرة المسوقين.

## طريقة العمل بعد الاستبدال
1. ارفع الملفات لنفس المسارات داخل المشروع.
2. نفّذ:
   ```bash
   npm install
   ```
3. تأكد من وجود:
   ```bash
   OPENAI_API_KEY=...
   OPENAI_MODEL=gpt-4.1-mini
   NEXT_PUBLIC_ADMIN_EMAILS=your-admin@email.com
   ```
4. افتح `/admin/inbox`
5. الصق رسالة أو ارفع TXT / ZIP
6. سيقوم النظام بـ:
   - حفظ الواضح تلقائيًا
   - إبقاء الغامض في المراجعة
   - تجاهل غير المهم

## مجموعات Firestore
هذه المجموعات لا تحتاج إنشاء يدوي. ستظهر عند أول كتابة:
- `fanar_requests`
- `fanar_listings`
- `fanar_inbox_entries`
- `fanar_extracted_items`

## قواعد منطقية مهمة في هذه النسخة
- الرسالة الصادرة منك لا تُحفظ كعرض وارد.
- الرسائل القصيرة والمتابعات تُتجاهل.
- إذا وُجد اسم بدون رقم يحفظ الاسم.
- إذا وُجد رقم بدون اسم يعتمد الاسم الافتراضي `مسوق - *****`.
- إذا لم يوجد اسم ولا رقم وبقي النص غير واضح، يتحول إلى `needs_review`.
